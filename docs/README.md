# Operational Documents

This directory contains operational notes, deployment records, and decisions that support the implementation.

For the intended behavior and architecture, start in `../specs/`.

## Documents

- `checklist_pendencias_implantacao.md`: deployment checklist and current readiness state.
- `troubleshooting.md`: local troubleshooting steps.
- `go_no_go.md`: demo acceptance criteria, rollback, and Go/No-Go notes.
- `chromadb_decision.md`: operational note for deferring ChromaDB.
- `accessibility_validation.md`: UX accessibility checklist and known residual risks.
- `http_api_contracts.md`: post-demo HTTP contracts for analysis, diary, and audio.
- `audio_privacy_policy.md`: privacy baseline for future audio input.
- `issue_estado_atual_aplicacao.md`: issue draft used to register the demo state.
- `issue_estado_atual_aplicacao_en.md`: English translation of the demo-state issue draft.
- `issue_ux_demo_readiness_streamlit.md`: issue draft for UX implementations already completed in the demo branch.
- `issue_demo_publica_controlada.md`: issue draft for the protected public demo deployment.
- `issue_landing_page_project.md`: issue draft for the project landing page.
- `../landing/index.html`: static trilingual landing page for EmpathyAI.
- `architecture.md`, `privacy_model.md`, `ethical_framework.md`: legacy operational summaries retained for quick reference.
